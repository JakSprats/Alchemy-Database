Alchemy-Database by JakSprats

Alchemy Database: A Hybrid Relational-Database/NOSQL-Datastore

Alchemy Database is a full RDBMS and a full NOSQL datastore in a single server

Alchemy Database is a lightweight SQL server that is built on top of the NOSQL datastore redis, w/ full support for redis data-structures/commands. Lua is deeply embedded and lua scripts can be run internally on Alchemy's data objects. Alchemy Database is also blazingly fast and extremely memory efficient.

  • Speed is achieved by being an event driven network server that stores ALL data in RAM and achieves disk persistence by using a spare cpu-core to periodically log data changes (i.e. no threads, no locks, no undo-logs, no disk-seeks, serving data over a network at RAM speed)
  • Storage data structures w/ very low memory overhead and data compression, via algorithms w/ insignificant performance hits, greatly increase the amount of data you can fit in RAM
  • Optimising to the SQL statements most commonly used in OLTP workloads yields a lightweight SQL server designed for low latency at high concurrency (i.e. mindblowing speed).

The Philosophy of Alchemy Database is that RAM is now affordable enough to be able to store ENTIRE OLTP Databases in a single machine's RAM (e.g. Wikipedia's English DB is 30GB and a Dell T610 w/ 32GB RAM costs $2100), as long as the data is made persistent to disk and can be replicated asynchronously over the wire. So Alchemy Database provides a non-blocking event-driven network-I/O-based relational-database, with very little memory overhead, that does the most common OLTP SQL statements amazingly fast and then throws in the NOSQL Data-store redis to create fantastic optimisation possibilities.

Install

1.) Download the code from git via "git clone git://github.com/JakSprats/Alchemy-Database.git"

1.) type "cd Alchemy=Database"

1.) type "make"

3.) run "./alchemy-server"
